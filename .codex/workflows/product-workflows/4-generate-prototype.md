# 4 - Generate Prototype

## 阶段目标

使用 `frontend-design` 技能，根据阶段 2 和阶段 3 的文档生成 HTML 高保真前端页面原型。

本阶段只生成原型，不重新发散需求，不重写前两阶段文档。

## 可参考资料

允许参考：

- `.codex/skills/frontend-design.md`
- 阶段 2 《系统概述与功能需求.md》
- 阶段 3 《非功能性需求与业务规则.md》
- `docs/system-management-srs.md`
- `docs/database-detail-design.md`

参考原则：

- 前两阶段文档是主输入。
- `docs` 只能用于补充字段、表结构、状态值和已有模块上下文。
- 如果 `docs` 与前两阶段确认结果冲突，以前两阶段确认结果为准。

## 必须执行

1. 读取 `.codex/skills/frontend-design.md`。
2. 提取所有页面、字段、筛选项、表格列、按钮、弹窗、状态、异常提示和交互。
3. 使用 HTML5 + Tailwind CSS CDN + 原生 JavaScript。
4. 严禁遗漏任何字段和操作按钮。
5. 必须体现基础交互示意，如弹窗显示隐藏、删除确认、按钮 hover、表单校验、Toast 提示。
6. 输出后必须提供覆盖自检清单。

## 输出要求

输出必须包含：

1. PRD 解析结果
2. HTML 完整代码
3. 原型覆盖自检

## 输出结束语

```markdown

```

## 停顿规则

阶段 4 完成后必须停止。

如果用户要求修改原型视觉或交互，可以在本阶段内调整。

如果用户提出新增字段、权限、状态或核心流程，必须回到阶段 1、2 或 3 重新确认。
