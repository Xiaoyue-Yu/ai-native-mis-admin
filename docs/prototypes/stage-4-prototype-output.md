# 阶段 4：系统管理模块用户管理原型输出

## PRD 解析结果

我已从 SRS 中识别到本次原型重点页面与功能：

1. 页面：系统管理 / 用户管理
   - 功能：用户分页查询、用户名/状态/部门筛选、高级筛选、用户列表展示、分页、空状态、加载状态。
   - 字段：`id`、`username`、`nickname`、`email`、`phone`、`avatar`、`sex`、`status`、`dept_id`、`remark`、`login_ip`、`login_time`、`create_time`、`update_time`、`create_by`、`roleIds`、`deleted`。
   - 操作：查询、重置、新增用户、编辑、删除、详情、启用/禁用、分配角色、重置密码、导入、导出、批量删除、刷新缓存。

2. 页面结构：企业级后台布局
   - 功能：左侧系统管理菜单、顶部用户信息、指标卡、筛选区、工具栏、表格区、分页区、弹窗层。
   - 权限：在页面中体现 `sys:user:list` 与 `sys:user:{action}` 权限标识规范。
   - 状态：启用、禁用、锁定、正常、异常、成功、失败、已逻辑删除提示。

3. 弹窗：新增用户弹窗
   - 字段：用户名、密码、昵称、邮箱、手机号、头像、性别、状态、部门、备注、角色集合。
   - 校验：必填项、用户名唯一性示意、密码强度/长度、邮箱格式、手机号格式、角色至少选择一项。
   - 交互：保存、取消、表单错误提示、Toast 成功/失败提示。

4. 风险操作弹窗
   - 场景：删除用户、批量删除、重置密码。
   - 规则：删除采用逻辑删除，仅更新 `deleted=1`；重置密码属于高风险操作。
