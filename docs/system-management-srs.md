## 1. 引言

### 1.1 目的

本文档依据 IEEE 830 软件需求规格说明书结构，定义 MIS 系统管理模块的功能需求、非功能需求、接口需求、数据需求、UML 模型和需求追踪关系，为后续设计、编码、测试、验收与维护提供统一依据。

本文档同时反映当前代码库的实现状态：系统目标范围覆盖用户管理、角色权限、组织架构、系统配置、数据字典、日志审计、系统监控等能力；当前仓库已优先实现用户新增、编辑、详情、逻辑删除、用户-角色关联及相关数据库基础表。

### 1.2 范围

系统管理模块是 MIS 平台的基础支撑模块，为业务系统提供统一的账号、权限、组织、配置、字典、日志和运行监控能力。模块采用前后端分离架构，后端基于 Spring Boot 3.2、MyBatis-Plus、MySQL、Redis 规划实现，前端基于 Vue 3、TypeScript、Element Plus、Pinia、Vite 实现。

## 2. 总体描述

### 2.1 产品视角

系统管理模块是 MIS 平台的基础模块，向其他业务模块提供统一的身份认证、用户资料、角色权限、部门组织、系统参数、数据字典、操作审计和运行监控服务。当前工程采用多模块 Maven 后端工程与独立前端工程：

| 层次 | 当前/规划组件 | 职责 |
| --- | --- | --- |
| 展示层 | `mis-admin-ui` Vue 3 应用 | 管理后台页面、表单、表格、交互状态 |
| 接口层 | `mis-system` Controller | HTTP 请求接收、参数校验、统一响应 |
| 业务层 | `mis-system` Service | 业务规则、事务、用户角色同步 |
| 数据层 | MyBatis-Plus Mapper、Flyway 脚本 | 数据访问、表结构迁移、逻辑删除 |
| 公共层 | `mis-common`、`mis-framework` | 统一响应、分页结果、基础实体、业务异常、全局异常 |

### 2.2 产品功能概要

系统目标功能由六大类组成：

| 功能域 | 目标能力 | 当前仓库状态 |
| --- | --- | --- |
| 用户管理 | 登录注销、用户 CRUD、状态管理、密码管理、导入导出 | 已实现新增、编辑、详情、逻辑删除、角色 ID 关联；分页列表、登录、重置密码、导入导出待实现 |
| 角色与权限管理 | 角色 CRUD、菜单/按钮/API/数据权限、用户角色关联、权限拦截 | 已有角色、菜单、关系表脚本和初始权限数据；业务接口待实现 |
| 组织架构管理 | 部门树、岗位、人员调动、组织可视化 | 已有部门表和初始树数据；业务接口待实现 |
| 系统配置管理 | 参数配置、数据字典、公告、主题配置 | 已有配置表、字典表和初始数据；业务接口待实现 |
| 日志与审计 | 操作日志、登录日志、异常日志、审计追踪 | 已有操作日志表；AOP 记录、查询和登录日志待实现 |
| 系统监控 | 在线用户、健康检查、资源监控、API 统计与限流 | 已有健康检查 Controller；完整监控能力待实现 |

### 2.3 用户类与特征

| 用户类 | 特征 | 主要诉求 |

| 普通管理员 | 负责特定部门或业务域 | 在授权范围内维护用户、部门、字典和配置 |

### 2.5 设计与实现约束

| 约束编号 | 约束 |
| --- | --- |
| CON-01 | Controller 层禁止直接操作数据库，必须调用 Service 层 |
| CON-02 | 禁止跳过 Service 层直接调用 Mapper 完成业务流程 |
| CON-03 | 权限标识格式应遵循 `sys:{module}:{action}`，禁止散落式硬编码 |
| CON-04 | 禁止使用 `SELECT *`，查询字段应明确或由 ORM 规范控制 |
| CON-05 | 禁止在代码中硬编码数据库连接信息和敏感凭据 |
| CON-06 | 后端系统实体类统一使用 `Sys` 前缀并继承 `BaseEntity` |
| CON-07 | 新增、修改、删除等写操作必须由事务保护 |
| CON-08 | API 响应必须使用统一 `Result<T>` 包装，分页使用 `PageResult<T>` |
| CON-09 | 数据库系统表统一使用 `sys_` 前缀，公共字段包含创建/更新时间、创建/更新人、逻辑删除标记 |
| CON-10 | 需求、设计、代码、测试之间应建立可追踪关系，进度通过文件系统持久化 |

### 3.5 业务规则

| 编号 | 规则 |
| --- | --- |
| BR-001 | 用户名必须唯一，新增用户时若用户名已存在应拒绝创建。 |
| BR-002 | 密码不得明文存储，必须使用 BCrypt 等不可逆算法加密。 |
| BR-003 | 用户删除采用逻辑删除，不应直接物理删除用户记录。 |
| BR-004 | 用户角色保存时应去除重复和空角色 ID。 |
| BR-005 | 编辑用户时不得修改用户名和密码，密码变更应使用单独的密码管理流程。 |
| BR-006 | 字典类型 `dict_type`、配置键 `config_key`、角色标识 `role_key` 应保持唯一。 |
| BR-007 | 菜单权限标识应遵循 `sys:{module}:{action}`。 |
| BR-008 | 禁用用户不得登录系统，锁定用户需由管理员解锁或按策略自动解锁。 |
